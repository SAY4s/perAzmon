# آزمون آنلاین تلگرام — راهنمای راه‌اندازی (نسخه‌ی چندآزمونه)

## ساختار پروژه

```
quiz_project/
├── quiz_bot.py            ← ربات تلگرام (بک‌اند) — نمره‌دهی، صف بررسی تشریحی
├── questions_editor.py    ← ابزار گرافیکی مدیریت آزمون‌ها (مستقل، آفلاین، بدون اتصال به ربات/سرور)
├── exams.json             ← فایل آزمون‌ها و سؤالات (با questions_editor.py ساخته/ویرایش می‌شود)
├── index.html             ← فرانت‌اند Mini App (لیست آزمون‌ها + فرم ثبت‌نام + آزمون)
└── README.md              ← همین فایل
```

⚠️ این نسخه از ساختار قبلی (`manage_questions.py` ترمینالی، `questions.json` تک‌آزمونه) متفاوت است.
سایت کاملاً استاتیک باقی مانده — **هیچ سرور یا دیتابیسی وجود ندارد**. همه‌ی منطق نمره‌دهی،
صف بررسی تشریحی، و پیام «خوش برگشتی» در خود ربات تلگرام (در حافظه‌ی RAM) انجام می‌شود.

## مرحله ۱ — نصب پیش‌نیازها

```bash
pip install pyTelegramBotAPI
```

برای استفاده از `questions_editor.py` نیاز به `tkinter` داری (در اکثر نصب‌های پایتون از قبل هست؛
در لینوکس اگر نبود: `sudo apt install python3-tk`).

---

## مرحله ۲ — ساخت ربات در BotFather

۱. به [@BotFather](https://t.me/BotFather) پیام بده، `/newbot` بزن.
۲. توکن را در `quiz_bot.py` قرار بده:
```python
BOT_TOKEN = "توکن واقعی اینجا"
```

⚠️ **توکن قبلی این پروژه در یک مکالمه فاش شده بود — حتماً از BotFather با دستور
`/revoke` یک توکن جدید بگیر و آن را جای توکن قدیمی بگذار.**

---

## مرحله ۳ — گرفتن Chat ID ادمین

به [@userinfobot](https://t.me/userinfobot) پیام بده و عدد chat id را در `quiz_bot.py` بگذار:
```python
ADMIN_CHAT_ID = "همون عدد"
```

## مرحله ۴ — میزبانی فرانت‌اند با HTTPS

تلگرام فقط HTTPS قبول می‌کند. توصیه می‌شود `index.html` و `exams.json` روی
**GitHub Pages** میزبانی شوند (HTTPS رایگان و آدرس ثابت). برای تست لوکال هم می‌توانی
از ngrok استفاده کنی (طبق راهنمای نسخه‌ی قبلی).

آدرس نهایی را در `quiz_bot.py` بگذار:

```python
WEBAPP_URL = "https://say4s.github.io/perAzmon/"
```

## مدیریت آزمون‌ها و سؤالات — با ابزار گرافیکی

```bash
python questions_editor.py
```

یک پنجره‌ی گرافیکی باز می‌شود که می‌توانی:
- **آزمون جدید بسازی** (با عنوان، توضیح، و تنظیمات نمره‌دهی مخصوص خودش)
- **برای هر آزمون به‌صورت جدا تنظیم کنی:**
  - نمره‌ی منفی برای پاسخ غلط (۰ یعنی بدون نمره‌ی منفی)
  - الزامی بودن پاسخ به همه‌ی سؤالات
- **سؤال چهارگزینه‌ای یا تشریحی** اضافه/ویرایش/حذف کنی
- ترتیب سؤالات را با دکمه‌ی بالا/پایین تغییر دهی
- با دکمه‌ی «ذخیره در exams.json» همه‌چیز را ذخیره کنی

⚠️ **این ابزار کاملاً مستقل است و به ربات یا هیچ سروری وصل نمی‌شود.**
بعد از ذخیره، خودت باید فایل `exams.json` تولیدشده را به‌جای نسخه‌ی قبلی
روی هاست (مثلاً GitHub Pages) جایگزین کنی.

## ساختار فایل `exams.json`

```json
[
  {
    "id": "general-knowledge",
    "title": "آزمون عمومی",
    "description": "توضیح کوتاه که در لیست آزمون‌ها نشان داده می‌شود",
    "settings": {
      "wrong_answer_penalty": 0.25,
      "require_all_answers": false
    },
    "questions": [
      {
        "id": 1,
        "type": "multiple_choice",
        "question": "متن سؤال",
        "choices": ["گزینه ۱", "گزینه ۲", "گزینه ۳", "گزینه ۴"],
        "correct_index": 1
      },
      {
        "id": 5,
        "type": "essay",
        "question": "متن سؤال تشریحی",
        "max_score": 5
      }
    ]
  }
]
```

- هر آزمون یک `id` یکتا دارد (خودکار از روی عنوان ساخته می‌شود).
- `settings.wrong_answer_penalty`: عددی که به ازای هر پاسخ غلط از نمره کسر می‌شود. صفر یعنی بدون نمره‌ی منفی.
- `settings.require_all_answers`: اگر `true` باشد، کاربر نمی‌تواند آزمون را بدون پاسخ به همه‌ی سؤالات تمام کند.
- سؤالات `multiple_choice`: دقیقاً ۴ گزینه دارند، `correct_index` از صفر شروع می‌شود.
- سؤالات `essay`: پاسخ متنی آزاد دارند و نمره‌شان (`max_score`) توسط ادمین، بعد از ارسال،
  دستی در چت ربات تعیین می‌شود.

## نحوه‌ی کار سایت (index.html)

سایت کاملاً استاتیک است:
1. لیست آزمون‌های موجود از `exams.json` نمایش داده می‌شود.
2. کاربر آزمون مورد نظر را انتخاب، فرم ثبت‌نام (نام، نام‌خانوادگی، کد ملی) را پر می‌کند.
3. سؤالات یکی‌یکی نمایش داده می‌شوند — با دکمه‌ی «سؤال قبلی» و «سؤال بعدی» می‌توان جلو و عقب رفت.
4. سؤالات تشریحی با کادر متنی نمایش داده می‌شوند.
5. در پایان، نتیجه (تخمینی، بدون نمره‌ی تشریحی) نشان داده می‌شود و کاربر با دکمه‌ی
   «ارسال نتیجه و پایان»، داده را با `tg.sendData()` به ربات می‌فرستد.

نمره‌ی نهایی (با اعمال نمره‌ی منفی طبق تنظیمات آزمون) توسط **خود ربات** دوباره محاسبه
می‌شود — سایت فقط یک تخمین برای نمایش فوری به کاربر می‌دهد.

---

## نحوه‌ی کار ربات (quiz_bot.py)

وقتی نتیجه از Mini App می‌رسد:
1. نمره را طبق تنظیمات همان آزمون (نمره‌ی منفی، اجباری بودن) دوباره و دقیق محاسبه می‌کند.
2. اگر سؤال تشریحی پاسخ داده شده باشد، آن را در صف بررسی قرار می‌دهد.
3. خلاصه را برای ادمین می‌فرستد (شامل تعداد صحیح/غلط/بی‌پاسخ و نمره).
4. به **خود کاربر** هم نتیجه را نشان می‌دهد — و اگر این آزمون را قبلاً هم
   داده باشد (در همین اجرای ربات)، نمره‌ی این‌بار را با دفعه‌ی قبل مقایسه
   می‌کند و بسته به بهتر یا بدتر شدن، پیام تبریک یا دلگرمی می‌فرستد.

### بررسی دستی سؤالات تشریحی

ادمین در همان چت بات دستور زیر را می‌زند:

```
/بررسی
```

ربات یکی‌یکی پاسخ‌های تشریحی در صف را (همراه با نام و کد ملی شرکت‌کننده) نشان می‌دهد
و از ادمین می‌خواهد یک عدد (نمره) ارسال کند. بعد از هر نمره، خودکار به پاسخ بعدی می‌رود.

⚠️ **نکته‌ی مهم درباره‌ی حافظه:** این ربات هیچ دیتابیسی ندارد. صف بررسی و
نمرات قبلیِ مقایسه‌شده، فقط در حافظه‌ی RAM ربات نگه داشته می‌شوند. اگر ربات
ری‌استارت شود (مثلاً به‌خاطر آپدیت سرور یا کرش)، این اطلاعات از بین می‌روند.
این یک انتخاب عمدی برای سادگی است.

## نکات امنیتی برای استفاده‌ی واقعی (Production)

- توکن ربات را عمومی نکن (در گیت‌هاب پابلیک قرار نده).
- اگر `index.html` را روی هاست عمومی (مثل GitHub Pages) می‌گذاری، توجه کن `exams.json`
  هم عمومی و قابل مشاهده خواهد بود — اگر سؤالات حساس‌اند، این روش مناسب نیست.
- `ADMIN_CHAT_ID` را با chat id واقعی خودت تنظیم کن.
