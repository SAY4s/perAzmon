# آزمون آنلاین تلگرام — راهنمای راه‌اندازی

## ساختار پروژه

```
quiz_project/
├── quiz_bot.py            ← ربات تلگرام (بک‌اند)
├── manage_questions.py    ← ابزار مدیریت سؤالات (مستقل، خط‌فرمان)
├── questions.json         ← فایل سؤالات (خودکار توسط manage_questions.py ساخته/ویرایش می‌شود)
├── index.html             ← فرانت‌اند Mini App (فرم ثبت‌نام + آزمون)
└── README.md              ← همین فایل
```

---

## مرحله ۱ — نصب پیش‌نیازها

```bash
pip install pyTelegramBotAPI
```

(کتابخونه‌ی `json` و `os` که توی کدها استفاده شده‌اند، جزو کتابخانه‌ی استاندارد پایتون هستند و نیازی به نصب ندارند.)

---

## مرحله ۲ — ساخت ربات در BotFather

۱. توی تلگرام به [@BotFather](https://t.me/BotFather) پیام بده
۲. `/newbot` بزن، یک اسم و یوزرنیم برای ربات انتخاب کن
۳. توکنی که می‌ده رو کپی کن و توی `quiz_bot.py` این خط رو پر کن:
```python
BOT_TOKEN = "اینجا توکن واقعی"
```

---

## مرحله ۳ — گرفتن Chat ID ادمین

۱. به ربات [@userinfobot](https://t.me/userinfobot) پیام بده
۲. عددی که بهت می‌ده (chat id) رو کپی کن
۳. توی `quiz_bot.py`:
```python
ADMIN_CHAT_ID = "همون عدد"
```

---

## مرحله ۴ — میزبانی فرانت‌اند (index.html) با HTTPS

⚠️ **نکته‌ی مهم:** تلگرام فقط آدرس‌های **HTTPS** را برای Mini App قبول می‌کند. آدرس `http://localhost` به‌طور مستقیم کار نمی‌کند.

### روش ساده برای تست لوکال — با ngrok

۱. یک سرور استاتیک لوکال روی پورت ۸۰۰۰ بالا بیار (همون پوشه‌ای که `index.html` و `questions.json` توشه):
```bash
cd quiz_project
python -m http.server 8000
```

۲. در یک ترمینال دیگر، با [ngrok](https://ngrok.com) این پورت رو عمومی (HTTPS) کن:
```bash
ngrok http 8000
```

۳. ngrok یک آدرس مثل `https://abc123.ngrok-free.app` می‌دهد. این آدرس را در `quiz_bot.py` بگذار:
```python
WEBAPP_URL = "https://abc123.ngrok-free.app"
```

> راه‌حل‌های جایگزین ngrok: Cloudflare Tunnel، localtonet، یا میزبانی نهایی روی GitHub Pages / Netlify / Vercel (که خودشان HTTPS رایگان می‌دهند و آدرس ثابت دارند — برای استفاده‌ی واقعی، توصیه می‌شود).

## مرحله ۵ — اجرای ربات

```bash
python quiz_bot.py
```

باید این خط را ببینی:
```
✅ ربات آزمون در حال اجراست...
```

این ترمینال را باز و در حال اجرا نگه دار — چون ربات با `polling` کار می‌کند (مدام از تلگرام سؤال می‌کند «چیزی برام اومده؟») و نیازی به webhook یا تنظیمات سرور اضافه ندارد.

---

## مرحله ۶ — تست کامل

۱. توی تلگرام به ربات خودت `/start` بزن
۲. روی دکمه‌ی **«📝 شروع آزمون»** که زیر باکس تایپ ظاهر می‌شود کلیک کن

   ⚠️ این دکمه باید از نوع *Keyboard Button* باشد (زیر چت، نه داخل پیام). در کد فعلی همین‌طور تنظیم شده — این یک محدودیت رسمی تلگرام است که فقط با این نوع دکمه، ارسال داده (`sendData`) به‌طور قابل‌اعتماد کار می‌کند.

۳. فرم ثبت‌نام را پر کن (نام، نام خانوادگی، کد ملی)
۴. آزمون را کامل کن
۵. در صفحه‌ی نتیجه، روی **«ارسال نتیجه و پایان»** بزن
۶. باید بلافاصله توی ترمینال پایتون اتفاقی ثبت شود و یک پیام خلاصه به `ADMIN_CHAT_ID` برسد

## مدیریت سؤالات

برای افزودن، ویرایش یا حذف سؤالات، **هیچ‌وقت مستقیم کد `index.html` یا `quiz_bot.py` را تغییر نده**. به‌جایش:

```bash
python manage_questions.py
```

یک منوی ساده باز می‌شود که می‌توانی:
- همه‌ی سؤالات را ببینی
- سؤال جدید اضافه کنی
- سؤال موجود را ویرایش کنی
- سؤال را حذف کنی
- یا یک‌جا فایل JSON دیگری را وارد (import) کنی

تغییرات مستقیم در فایل `questions.json` ذخیره می‌شود و دفعه‌ی بعد که کاربر آزمون را باز کند، نسخه‌ی جدید را می‌بیند — بدون نیاز به تغییر کد یا ری‌استارت چیزی (چون `index.html` هر بار از نو فایل را می‌خواند).

---

## ساختار فایل `questions.json`

هر سؤال این شکل را دارد:

```json
{
  "id": 1,
  "question": "متن سؤال اینجا",
  "choices": ["گزینه ۱", "گزینه ۲", "گزینه ۳", "گزینه ۴"],
  "correct_index": 1
}
```

- `id`: شناسه‌ی یکتای عددی (خودکار توسط `manage_questions.py` تخصیص داده می‌شود)
- `choices`: دقیقاً باید ۴ گزینه باشد
- `correct_index`: شماره‌ی گزینه‌ی صحیح، از **صفر** شروع می‌شود (یعنی برای گزینه‌ی اول `0`، دوم `1`، سوم `2`، چهارم `3`)

## نکات امنیتی برای استفاده‌ی واقعی (Production)

- توکن ربات را عمومی نکن (مثلاً در گیت‌هاب پابلیک قرار نده)
- اگر `index.html` را روی یک هاست عمومی (مثل GitHub Pages) قرار می‌دهی، توجه داشته باش که `questions.json` هم عمومی و قابل مشاهده خواهد بود — اگر سؤالات حساس هستند، باید آن‌ها را پشت یک بک‌اند با احراز هویت قرار دهی به‌جای فایل استاتیک
- `ADMIN_CHAT_ID` را با چت آیدی واقعی خودت (نه کسی دیگر) تنظیم کن
